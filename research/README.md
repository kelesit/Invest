# research/

研究日志的根目录。层级：

```
research/
  README.md                    # 本文件：全局工作流和反模式
  TEMPLATE.md                  # 实验预注册模板
  proj_<name>/
    README.md                  # 项目文档：主问题、假设树、工作信念、结束条件
    exp_NNN_<slug>.md          # 具体实验
```

## 层级职责

每一层必须装别的层装不下的东西，否则不该存在：

| 层 | 装什么 | 不装什么 |
|---|---|---|
| `CLAUDE.md` | 项目愿景 + 已完结项目的回顾段 | live 项目的细节 |
| `docs/beliefs.md` | 跨项目、高 bar 的 anchors & lessons | 单项目的工作假设 |
| `research/proj_*/README.md` | 主问题、假设树、项目局部工作信念、结束条件 | 单个实验的原始数据 |
| `research/proj_*/exp_*.md` | 单次实验的预注册 + 结果 + 解读 | 跨实验的归纳 |

项目完结时：`proj_*/README.md` 被提炼成一段话搬进 `CLAUDE.md` 路线图，原 README 留在原地做历史记录。

## 实验工作流（不要跳步）

1. **开实验前**，复制 `TEMPLATE.md` → `proj_*/exp_NNN_xxx.md`，把前 6 节（问题 → 测量噪音估计）**全部填完**
2. 填不出"证伪条件"就说明你不清楚自己在测什么，停下来想清楚再跑
3. 填不出"测量噪音估计"就先跑那个——在不知道误差范围前看任何指标都没意义
4. 跑完实验，填 Result / Interpretation / Beliefs 更新
5. **Interpretation 必须是关于市场的推论，不是关于代码的修改。** 如果唯一的收获是"参数 X=Y 更好"，大概率这个实验什么都没教你——诚实地写下来
6. 如果实验改变了项目层的假设树，更新 `proj_*/README.md` 的对应节点（活枝 → 死枝，或反之）
7. 如果结论够硬到能进 `docs/beliefs.md`，在那里动手，并在实验文件里链接过去

## 反模式（看到就停）

- 改参数 → IC 涨了 → 开下一个实验。这是在 IC 上做梯度下降，不是在研究
- 实验跑完才补写 hypothesis。写在后面一定会 rationalize 成胜利
- 没有 baseline / 测量噪音就讨论"信号存在"
- 连续几个实验都没改动项目 README 的假设树，也没动 `docs/beliefs.md`。说明你在原地踏步
- 在 Claude 的帮助下让 Claude 起草问题 / 先验 / 证伪条件 / 解读 / 下一步。这些是决策，决策必须你做——否则实验结果就是一份你读不懂的答卷

## 关于工具分工

- **Claude 可以做**：样板代码、数据加载、可视化、工具函数、数值实现、挑战你写的设计
- **你必须做**：问题定义、标签选择、指标选择、证伪条件、结果解读、下一步决策

判断标准：**如果做错了会让你学到东西，那它必须你做。**
