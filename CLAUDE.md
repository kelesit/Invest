# 项目愿景

最终目标是建立一个能承载任意因子、任意品种、任意市场的量化研究和交易系统。CTA 趋势跟踪是入口，下一步转向股票截面（多因子选股）。

## 当前阶段

CTA 基础研究已完成（项目一到四），正在转向股票截面方向。

## 多因子选股链路

```
数据获取 → 特征工程 → 标签构造 → 模型训练 → 组合优化 → 执行 → 监控
```

### 1. 数据获取
- 量价数据：OHLCV、分钟线
- 财务数据：财报、分析师预期（Tushare/AKShare）
- 另类数据：新闻/公告 NLP、行业专有数据

### 2. 特征工程
- 对原始数据做系统性变换：原始字段 × 变换算子 × 时间窗口 = 几百个特征
- 特征矩阵：每行 = 一只股票某天的快照，每列 = 一个特征
- 所有特征转截面百分位排名（消除量纲差异）
- **核心：把对市场的理解翻译成数学表达**

### 3. 标签构造
- 标签定义了问题本身，换标签比换模型提升更大
- 未来 N 天收益率（方向）、残差收益率（纯 alpha）、财报后异常收益（事件驱动）、最大回撤（风控）
- 优先用去掉市场/行业因子后的残差收益率

### 4. 模型训练
- 主流：LightGBM/XGBoost，宁欠拟合不过拟合
- 防过拟合五道防线：
  1. Purged Time-Series CV（purge + embargo，防时间泄漏）
  2. 控制模型复杂度（浅树、强正则、低学习率）
  3. 特征筛选（IC 检验 → 去共线性 → feature importance）
  4. 多重检验校正（Benjamini-Hochberg FDR）
  5. 样本外最终验证（留出数据全程不碰）
- 模型每天/每周重训

### 5. 组合优化
- 模型输出打分 → 买入排名前 N 只，卖出排名后 N 只
- 约束：行业暴露、风格暴露、换手率
- 在信号强度、交易成本、风险约束之间求最优解

### 6. 执行
- 交易成本模型：根据订单大小、流动性、波动率动态估算
- 算法拆单：TWAP/VWAP
- 周频/月频调仓时执行层差异不大

### 7. 监控
- 因子 IC 持续追踪，检测因子失效
- 回撤归因：定位哪个因子、哪个行业出问题

### 个人发力点
```
特征工程 ≈ 标签构造 >> 数据的创造性处理 > 组合优化 > 执行
```

### 从 CTA 阶段学到的可迁移认知
- 低相关 ≠ 有价值，多因子组合需要每个因子本身有正期望收益
- 防过拟合比提升信号更重要
- ATR sizing 等风控机制存在双重调整问题，风控层不是越多越好

### 项目路线图

**项目一：回测可信度验证 — 已完成**
问题：我怎么知道回测结果不是碰巧好的？
- 数据分 2019-2023（训练）和 2024-2026（验证），看样本外是否还赚钱
- 参数扫描：lookback 从 5 到 120 全扫一遍，画热力图，看是普遍赚钱还是只有某个参数赚
- 随机基准：生成 1000 条随机信号跑回测，看策略排第几
- 产出：一套验证工具，之后每研究一个新信号都能用

**项目二：信号研究框架 — 已完成**
问题：从价格时间序列能提取出哪些趋势因子？它们之间有多少独立信息？能否组合提升？
- 实现 10 种趋势信号（动量、均线、通道突破、MACD、回归斜率、ADX 等）
- 统一评估：信号×品种夏普热力图、信号相关性矩阵、权益曲线叠加
- 市场环境分析：趋势/震荡 × 高/低波动四种环境下的表现差异
- 信号组合：等权、逆波动率、低相关性三种组合方式对比
- 核心发现：10 个信号相关性 0.5-0.9（共享同一数据源——价格），组合不如最佳单信号，价格时间序列已被充分挖掘
- 产出：对趋势度量的完整认知、可扩展的信号库（`cta/signals.py` + `SIGNAL_REGISTRY`）

**项目三：风控与组合优化 — 已完成**
问题：风控能否改善组合表现？不同风控机制各有什么代价？
- 追踪止损：夏普降了（0.91→0.67）但回撤从 -9.5% 降到 -6%
- 波动率择时 & 组合波动率目标：维持恒定风险暴露
- 逆波动率权重 & 风险平价：考虑/不考虑相关性的权重分配
- 核心发现：止损的价值在回撤不在夏普；ATR sizing 本身已是风控，额外风控层存在双重调整问题；风险平价在 4 品种时不稳定
- 产出：风控模块（`cta/risk.py`）、组合优化模块（`cta/portfolio.py`）

**项目四：多因子融合（carry + 趋势） — 已完成**
问题：怎么引入与价格趋势低相关的数据源？
- 用 4 个品种的期限结构数据实现时间序列 carry 因子（近月 vs 远月价差）
- 验证了"不同数据源 = 低相关"：carry-趋势相关性 0.03-0.16，远低于趋势间的 0.63
- 但 carry 在 4 个品种上单独亏钱（Sharpe -1.30），组合后拖累趋势表现
- 核心教训：低相关 ≠ 有价值，多因子组合需要每个因子本身有正期望收益；时间序列 carry 在少品种上不可用，需要 10+ 品种的截面策略
- 产出：carry 信号（`cta/signals.py`）、数据加载（`cta/data_loader.py` 的 `load_carry_data`）、分析 notebook

**项目五：股票截面多因子 — 下一步**
问题：能否用 ML pipeline 在股票市场构建有效的多因子选股策略？
- 接入 A 股/美股股票数据（量价 + 财务）
- 批量特征生成 → LightGBM → purged CV 验证
- 最小版本：50 个量价因子 + 月度调仓
- 产出：完整的多因子选股 pipeline

## 工作方式

- 环境管理用 uv
- 数据源是 Databento，API key 在 .env 中
- 美股市场先行，之后加入 A 股
- 追求最小可用系统，逐步迭代，不过度设计
- 解释理论或写代码时，必须区分"理论上正确的做法"和"因数据/工具限制做的近似"。先讲理论，再讲近似及其误差。绝不把近似方案直接当作正确理论呈现——用户是初学者，会把第一次听到的说法当作真理
- 留在项目里的 Python 代码要有强复用性；学习探索向的分析可以用 notebook 记录
- **批判性诚实原则**：结果不好就是不好，不找借口。不能用"极端行情"、"样本不够"、"近似误差"等外部原因为失败的设计开脱。先承认失败，再诊断根本原因，再提改进方向。量化研究里掩盖失败比失败本身更危险——它会让错误的设计继续推进，浪费时间甚至真实资金
